Introduction to NestJS

What is NestJS?

So, first things first — what even is NestJS? In simple words, it's a progressive Node.js framework designed for building efficient and scalable server-side applications. Think of it as the grown-up, well-organized version of your typical Node.js backend.

  • Written in TypeScript and heavily inspired by Angular's design patterns and architecture.
  • Uses a modular architecture — everything is neatly grouped into modules, making large codebases much easier to manage.
  • Sits on top of Node.js (with Express under the hood by default), but layers a solid, structured system on top.
Note: If you've used Angular before, NestJS will feel very familiar — decorators, modules, dependency injection… all those patterns carry straight over to the backend.

Why Do We Even Need NestJS?

Honestly, when I first heard about NestJS I thought — "we already have Express, why bother?" But once you work on a bigger project, you start to see the problem. Things get messy, fast. That's exactly where NestJS comes in.

  • It simplifies backend development by applying modern, well-established architectural patterns right from the start.
  • Provides a structured, opinionated way to build applications that are both scalable and easy to test.
  • Directly solves the limitations of traditional Express apps — mainly the total lack of enforced structure as your codebase grows.
🎙️ Interview Question

NestJS vs ExpressJS — What's the Difference?

This one comes up a lot in interviews, so it's worth knowing cold.

ExpressJS is described as an "unopinionated, minimalist web framework." The key word is unopinionated — no enforced structure, you organize things however you want. That's great for small apps or quick prototypes, but for large-scale team projects? That freedom becomes a liability.

NestJS is the opposite — it's opinionated. It enforces a modular structure, and that's a feature, not a drawback. It also ships with a ton of built-in capabilities:

Feature ExpressJS NestJS
Language JavaScript TypeScript
Project Structure None enforced Modular & enforced
Dependency Injection Manual / not built-in Built-in system
Scalability Needs extra setup Designed for scale
GraphQL Manual integration Dedicated module
Microservices Not built-in First-class support
WebSockets Via external libs Built-in Gateway
Testing Manual setup Built-in test utilities
Learning Curve Low Medium (TypeScript needed)
💡 Interview Answer: "ExpressJS is unopinionated and minimal — great for small apps. NestJS is opinionated and structured, making it the better pick for large, scalable, team-based projects. It brings TypeScript, built-in dependency injection, and ready-made modules for GraphQL, WebSockets, and microservices — all out of the box."

Key Benefits of NestJS

  • Full TypeScript support — catch bugs at compile time, not in production at 2am.
  • Built-in Dependency Injection (DI) — cleanly manage and share services across your whole app.
  • Easy database integration — works seamlessly with TypeORM, Prisma, Mongoose, and more.
  • WebSockets & GraphQL — first-class support with dedicated modules for both.
  • Microservices support — built with distributed, service-oriented architecture in mind.
  • Scalable and maintainable codebase — the module system keeps things organized as the project grows.
  • Active and growing community — strong docs, packages, and community support to lean on.

02

Installation & Project Setup

Step 1 — Install Node.js

Before anything NestJS-related, we need Node.js on our machine. Head over to the official site and grab the LTS (Long-Term Support) version — it's the most stable one and the safest choice for any serious project.

🔗 Download: https://nodejs.org/en/download

⚠️ Important during installation: Just click Next → Next → Next → Finish. Do NOT check any extra checkboxes that appear during the setup — especially the one that offers to install additional tools/Chocolatey. We don't need those.

But Wait — Why Do We Need Node.js?

Here's the thing — JavaScript was originally built to run only inside browsers. That was fine for making web pages interactive, but it meant JS couldn't do anything on the server side.

Node.js changed all of that. It's a runtime environment that lets JavaScript run outside the browser — on your machine, on a server, basically anywhere. That's what made building backend apps with JavaScript (and TypeScript) possible in the first place.

💡 One-liner to remember: Node.js took JavaScript out of the browser and gave it a life beyond the frontend.

Once installed, open your terminal and verify it worked:

terminal 
# Check your Node.js version — should print something like v20.x.x
node -v

Step 2 — npm Comes Free with Node

Here's a nice bonus — when you install Node.js, npm (Node Package Manager) gets installed automatically alongside it. You don't need to do anything extra.

🎙️ Interview Question

What is npm and why do we need it?

npm is the default package manager for Node.js. Think of it like an app store, but for JavaScript libraries and tools. Instead of manually downloading and linking third-party code, you just run one command and npm handles everything — downloading the package, managing its version, and tracking it in your project.

  • Install packages — pull in any library from the npm registry with a single command.
  • Manage dependencies — keeps track of exactly which packages and versions your project needs via package.json.
  • Run scripts — lets you define and run commands like npm run start or npm run build.
  • Share your own packages — you can publish your own npm packages for others to use too.
terminal 
# Verify npm is installed — should print something like 10.x.x
npm -v
Note: If both node -v and npm -v print version numbers without errors, you're all set and ready for the next step.

Step 3 — Install the NestJS CLI

The NestJS CLI is a command-line tool that lets you scaffold new projects, generate files, and run your app — all from the terminal. We install it globally so it's available anywhere on your machine.

terminal 
# -g means "global" — installs the CLI system-wide, not just for one project
npm i -g @nestjs/cli

Step 4 — Create Your First NestJS Project

Now let's actually create a project. Follow these steps one by one:

  • 1
    Navigate to your projects folder

    Open your terminal and cd into wherever you keep your projects.

  • 2
    Run the project creation command

    Replace project-name with whatever you want to call your app.

  • 3
    Select npm as the package manager

    The CLI will ask "Which package manager would you ❤️ to use?" — choose npm.

  • 4
    Enter the project and open VS Code

    Once the setup finishes, jump into your new project folder and open it in your editor.

  • 5
    Start the development server

    Run the start command and then open your browser to check it's working.

terminal — full setup flow 
# 1. Move into your projects folder (adjust the path to yours)
cd projectfolder

# 2. Tell the NestJS CLI to scaffold a new project
nest new project-name

# 3. When prompted, select "npm" as your package manager
#    (use arrow keys to navigate, Enter to confirm)

# 4. Once created, enter the new project directory
cd project-name

# 5. Open the project in VS Code
code .

# 6. Start the development server
npm run start
💡 It's working! Once the server starts, open your browser and go to http://localhost:3000/ — you should see "Hello World!" staring back at you. That's your NestJS app running live.
